# CLI Contract: v0.2 Authentication Foundation

**Feature**: `002-auth-foundation`  
**Date**: 2026-04-26  
**Status**: Complete

---

## Overview

v0.2 replaces the hand-rolled `os.Args` dispatch from v0.1 with a cobra + fang command tree. This document defines the complete CLI contract for `rook-cli` as of v0.2, including all subcommands, flags, exit codes, and output format.

---

## Binary: `rook-cli`

### Root Command

```
rook [flags] [subcommand]
```

**Default behaviour** (no subcommand): Opens the local launcher TUI (stash file list + action menu). No auth required. Requires no network access.

**Flags** (managed by fang):

| Flag | Short | Description |
|---|---|---|
| `--version` | `-v` | Print version string and exit 0 |
| `--help` | `-h` | Print usage and exit 0 |

**Exit codes**:

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | Unrecoverable error (message printed to stderr by fang) |

---

### `rook auth`

```
rook auth [flags]
```

**Description**: Authenticate with a configured rook-server using SSH key challenge-response. Scans `~/.ssh/` for available private keys, prompts for selection (or uses `--key`), fetches a challenge, signs it, submits for verification, and displays confirmation on success. The session token is held in-memory for the process lifetime.

**Flags**:

| Flag | Description |
|---|---|
| `--key <path>` | Path to SSH private key; bypasses key discovery and selection prompt |

**Interactive behaviour**:
- Multiple keys found: presents a Bubble Tea list of available keys (filename, type, fingerprint); user selects with arrow keys + enter
- Single key found: skips prompt, prints informational message, proceeds with that key
- No keys found: prints error directing user to generate a key or use `--key`; exits 1

**Output** (stdout):
```
✓ Authenticated as <user_id>
  Session token valid for 1 hour
```

**Error output** (stderr, via fang):

| Condition | Message |
|---|---|
| No keys in `~/.ssh/` | `rook: no SSH keys found in ~/.ssh/ — generate one with ssh-keygen or specify a key with --key` |
| Key file not found | `rook: key file not found: <path>` |
| Nonce expired | `rook: authentication failed — challenge expired; run rook auth again` |
| Signature mismatch | `rook: authentication failed — signature did not match the registered public key` |
| Key not registered | `rook: authentication failed — public key not registered with the server` |
| Server unreachable | `rook: cannot reach server at <address>: <underlying error>` |

**Exit codes**:

| Code | Condition |
|---|---|
| `0` | Authentication succeeded |
| `1` | Any error condition listed above |

---

### `rook auth status`

```
rook auth status
```

**Description**: Stub. Displays current session status. In v0.2, sessions are in-memory only and are always absent when this command is invoked as a fresh process.

**Output** (stdout):
```
no active session
```

**Exit codes**:

| Code | Condition |
|---|---|
| `0` | Always (stub) |

---

### `rook auth logout`

```
rook auth logout
```

**Description**: Stub. Clears the current session. In v0.2, there is no persistent session to revoke; the command is correct and complete as a stub.

**Output** (stdout):
```
logged out
```

**Exit codes**:

| Code | Condition |
|---|---|
| `0` | Always (stub) |

## Config File

Location: `$XDG_CONFIG_HOME/rook/config.json` (fallback: `~/.config/rook/config.json`)

Relevant fields for v0.2:

| Field | Type | Purpose |
|---|---|---|
| `servers[].address` | `string` | Base URL of the rook-server to authenticate against |
| `servers[].alias` | `string` | Human-readable label (displayed in key selection prompt) |


## Backward Compatibility

v0.2 replaces the hand-rolled v0.1 flag dispatch. The following v0.1 behaviours are preserved or superseded:

| v0.1 behaviour | v0.2 behaviour |
|---|---|
| `rook --version` → prints version | `rook --version` → unchanged (managed by fang) |
| `rook --help` → prints help | `rook --help` → unchanged (managed by fang) |
| `rook` (no args) → prints version + config error message | `rook` (no args) → opens local launcher TUI |
| No subcommands | `rook auth`, `rook auth status`, `rook auth logout` added |
