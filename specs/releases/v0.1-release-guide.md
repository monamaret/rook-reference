## Release Naming Convention

Rook releases follow the pattern:

```
rook vMAJOR.MINOR[.PATCH]
```

| Segment | Rule |
|---------|------|
| **MAJOR** | Incremented for architectural breaks or scope resets (post-PoC only). |
| **MINOR** | Incremented for each planned product release (v0.1, v0.2, …). Matches the milestone PRD number. |
| **PATCH** | Reserved for hotfixes on a released minor. Created from a `hotfix/vX.Y.Z-<slug>` branch. |

## 1. Overview

v0.1 establishes the foundational structure of the Rook project: the `rook-cli` and `rook-server` Go module layouts, a shared `go.work` workspace, CI pipelines, local development tooling, and the XDG-compliant configuration file schema that all subsequent releases depend on. No user-facing product features are delivered. The outcome is a working skeleton that any contributor can clone, build, and run locally — producing two working binaries (`rook-cli` and `rook-server-cli`) that print their version and exit cleanly. The CI pipeline gates every PR to `main` on lint, test, and build from day one, preventing lint debt from accumulating before application code is written.

**Note — release.yml binary jobs at v0.1:** The `release.yml` workflow's Cloud Run deploy jobs require GCP Workload Identity Federation secrets (`GCP_WORKLOAD_IDENTITY_PROVIDER`, `GCP_SERVICE_ACCOUNT`) and an Artifact Registry repository. ROOK_CONFIG_PATH=/tmp/no-such-config.json ./dist/rook-cli

cd rook-cli && go test ./... -race -count=1
# expect: 14 tests PASS in internal/config

cd rook-cli && go test ./internal/config/... -v -run TestConfigPath

## 8. Scope Boundaries

The following items are explicitly deferred from this release:

- Authentication or identity — no user sessions, no SSH keypair generation (v0.2+)
- Any running services — no gRPC or HTTP servers started
- TUI or interactive CLI screens — `rook-cli` exits after printing version/help (v0.2+)
- Firestore or cloud infrastructure — no GCP resources provisioned
- Docker or Cloud Run deployment configuration (Dockerfiles deferred; Cloud Run deploy jobs in `release.yml` will fail until provisioned)
- `rook-docs/` Hugo site — directory exists in repo but contains no Go code or Hugo content
- `rook-server` services beyond the admin CLI stub — no auth, user, stash, or messaging services
- Admin subcommands in `rook-server-cli` — `user`, `space` deferred to v0.3+
- Project constitution (`.specify/memory/constitution.md`) — currently an unfilled template; must be authored before v0.2

See [PRD001 §6 — Out of Scope](../product/PRD001-rook-overview-v1.0.md#6-out-of-scope) for the global exclusion list.
